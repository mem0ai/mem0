# ADR-005: Auto-descoberta de conexão MCP via endpoint JSON

> **Idioma:** escreva todo o conteúdo deste arquivo em **português brasileiro (PT-BR)**.

## Status

Aceito

## Data

2026-06-15

## Contexto

O PRD exige um **endereço local de auto-descoberta** que permita aos agentes obter os dados necessários para **configurar sozinhos** a conexão MCP, eliminando configuração manual. O OpenMemory já expõe um servidor FastAPI; a rota MCP é `/mcp/{client_name}/sse/{user_id}` (SSE) e há também transporte HTTP.

## Decisão

Expor um **endpoint HTTP GET** no FastAPI existente que retorna um **JSON de configuração MCP pronta**: transporte (SSE/HTTP), URL base do servidor na rede local e o template de rota com os campos a preencher (`client_name`, hostname no slot `user_id`, `project`). O agente injeta esse JSON diretamente no seu cliente MCP. O conteúdo é o mínimo para autoconfiguração programática.

## Alternativas Consideradas

### Alternativa 1: Descritor + script de setup

- **Descrição**: Além do JSON, servir um script/instruções que configura o cliente automaticamente.
- **Prós**: Mais conveniente para humanos.
- **Contras**: Mais código a construir e manter; dependente de ambiente/cliente.
- **Por que rejeitada**: Excesso de escopo para o MVP; o JSON cobre a autoconfiguração programática (YAGNI).

### Alternativa 2: Caminho padronizado `.well-known`

- **Descrição**: Servir o mesmo JSON em um caminho convencional (ex.: `/.well-known/mcp`).
- **Prós**: Convenção previsível.
- **Contras**: Nenhuma diferença de conteúdo; convenção ainda não consolidada para MCP.
- **Por que rejeitada (parcialmente)**: Mantida como opção de caminho; a decisão é sobre **conteúdo** (JSON pronto), e o caminho exato pode ser definido na implementação.

## Consequências

### Positivas

- Onboarding de agente sem configuração manual.
- Reusa o FastAPI já existente; baixo custo de implementação.

### Negativas

- O agente ainda precisa **conhecer o endereço de descoberta** na primeira vez (bootstrap mínimo).

### Riscos

- **Bootstrap inicial**: como o agente descobre o endereço na primeira conexão; mitigar com endereço/porta padrão documentados e, em fase futura, anúncio na rede local.

## Notas de Implementação

- Adicionar uma rota GET no FastAPI do OpenMemory que monta o JSON a partir da config de runtime (host/porta, transportes habilitados).
- Incluir no JSON: `transport`, `base_url`, `route_template`, e os nomes dos campos esperados (`client_name`, `user_id`=hostname, `project`).
