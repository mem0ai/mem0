# MiroFish Integration

[MiroFish](https://github.com/666ghj/MiroFish) is an open-source swarm intelligence engine that simulates thousands of autonomous agents with independent personalities to predict real-world outcomes. It runs a five-stage pipeline — graph building, environment setup, simulation, report generation, and deep interaction.

Mem0 plugs into this pipeline as a memory layer so every agent can persist and recall memories across simulation stages with per-agent isolation.

<Info>
  **Prerequisites**
  - Mem0: `pip install mem0ai`, get your API key from [Mem0 Platform](https://app.mem0.ai)
  - MiroFish: cloned and set up per [MiroFish README](https://github.com/666ghj/MiroFish)
  - For OSS mode: `OPENAI_API_KEY` for LLM-powered memory extraction
</Info>

## Why Mem0 with MiroFish

MiroFish ships with Zep Cloud for memory. Mem0 offers a complementary path with distinct advantages:

| | Zep Cloud | Mem0 Platform | Mem0 OSS |
|---|---|---|---|
| **Self-hosting** | No | No | Yes (full control) |
| **Per-agent memory isolation** | Flat graph | Native `agent_id` scoping | Native `agent_id` scoping |
| **Node/memory limits** | Free tier capped | Usage-based | Unlimited |
| **Knowledge graph** | Built-in | Managed | Neo4j via `graph_store` |
| **Setup complexity** | API key only | API key only | Requires vector store + LLM |
| **Automatic entity extraction** | Built-in | Built-in | Via LLM + graph store |

<Tip>
  For the closest feature parity with Zep's knowledge graph, use **Mem0 OSS with Graph Memory** (Neo4j). This gives you automatic entity/relationship extraction. See the [Advanced: Graph Memory](#advanced-graph-memory-with-neo4j) section.
</Tip>

## How Mem0 Maps to MiroFish's Pipeline

MiroFish runs a five-stage workflow. Here's where Mem0 fits in each stage:

### Stage 1: Graph Building — Ingest Seed Documents

MiroFish ingests seed materials (news, reports, stories) to build background knowledge. With Mem0, each document chunk becomes a searchable memory.

<Tabs>
  <Tab title="Platform">
```python
from mem0 import MemoryClient

client = MemoryClient()
project_id = "mirofish_housing_sim"

# Ingest seed documents as background knowledge
seed_docs = [
    "The city council proposed a new zoning reform allowing higher density housing. "
    "Mayor Chen expressed strong support, citing a 40% housing shortage.",
    "Local homeowners association president Wang opposes the reform, "
    "arguing it will decrease property values by 15-20%.",
]

for doc in seed_docs:
    client.add(
        [{"role": "user", "content": doc}],
        user_id=project_id,
        metadata={"stage": "graph_building", "source": "seed_document"}
    )
```
  </Tab>
  <Tab title="OSS">
```python
from mem0 import Memory

config = {
    "llm": {
        "provider": "openai",
        "config": {"model": "gpt-4o-mini"}
    }
}
memory = Memory.from_config(config)
project_id = "mirofish_housing_sim"

# Ingest seed documents as background knowledge
seed_docs = [
    "The city council proposed a new zoning reform allowing higher density housing. "
    "Mayor Chen expressed strong support, citing a 40% housing shortage.",
    "Local homeowners association president Wang opposes the reform, "
    "arguing it will decrease property values by 15-20%.",
]

for doc in seed_docs:
    memory.add(
        doc,
        user_id=project_id,
        metadata={"stage": "graph_building", "source": "seed_document"}
    )
```

</Tabs>

### Stage 2: Environment Setup — Enrich Agent Profiles

Before simulation, MiroFish generates agent profiles. Use Mem0 to retrieve relevant context for each agent's persona.

```python
results = client.search(
```

### Stage 3: Simulation — Track Agent Activities Per Round

During simulation, agents perform actions (post, comment, like, follow). Each action is stored as a memory with the agent's own `agent_id` for isolation.

<Tabs>
  <Tab title="Platform">
```python
# Round 1: Agents post opening statements
client.add(
    [{"role": "user", "content": "Posted: 'This reform will create 10,000 new housing units by 2028.'"}],
    user_id=project_id,
    agent_id="mayor_chen",
    metadata={"action": "CREATE_POST", "platform": "twitter", "round": 1}
)

client.add(
    [{"role": "user", "content": "Posted: 'Our property values will plummet! #StopTheReform'"}],
    user_id=project_id,
    agent_id="wang_homeowner",
    metadata={"action": "CREATE_POST", "platform": "twitter", "round": 1}
)

# Round 2: Agents interact with each other
client.add(
    [{"role": "user", "content": "Commented on wang_homeowner's post: 'The data does not support your claims.'"}],
    user_id=project_id,
    agent_id="mayor_chen",
    metadata={"action": "CREATE_COMMENT", "platform": "twitter", "round": 2}
)

# Mid-simulation recall: agent reads own memory before deciding next action
mayor_memories = client.search(
    "What have I posted about housing reform?",
    filters={"AND": [{"user_id": project_id}, {"agent_id": "mayor_chen"}]}
)
```
  </Tab>
  <Tab title="OSS">

<Note>
  **Per-agent isolation**: Each agent only sees its own memories when `agent_id` is provided. This prevents cross-contamination — agent Wang cannot accidentally recall Mayor Chen's internal reasoning.
</Note>

### Stage 4: Report Generation — Semantic Retrieval for Analysis

After simulation completes, MiroFish's ReportAgent queries memory to generate prediction reports.

<Tabs>
  <Tab title="Platform">
```python
# Broad retrieval: get all simulation memories
all_memories = client.get_all(filters={"user_id": project_id})
print(f"Total memories across all agents: {len(all_memories['results'])}")

# Targeted search: find opposition arguments
opposition = client.search(
    "arguments against the housing reform",
    filters={"user_id": project_id},
    top_k=10
)
for r in opposition["results"]:
    print(f"  - {r['memory']}")

# Agent-specific search: what did a specific agent do?
wang_actions = client.search(
    "All actions and posts",
    filters={"AND": [{"user_id": project_id}, {"agent_id": "wang_homeowner"}]}
)
```
  </Tab>
  <Tab title="OSS">
```python
all_memories = memory.get_all(user_id=project_id)
print(f"Total memories across all agents: {len(all_memories['results'])}")

opposition = memory.search(
    "arguments against the housing reform",
    user_id=project_id,
    limit=10  # OSS uses top-level kwargs
)
for r in opposition["results"]:
    print(f"  - {r['memory']}")

wang_actions = memory.search(
    "All actions and posts",
    user_id=project_id,
    agent_id="wang_homeowner"  # OSS uses top-level kwargs
)
```

### Stage 5: Deep Interaction — Chat with Post-Simulation Memory

After reports are generated, users can chat with agents or the ReportAgent. Mem0 provides the memory context for these conversations.

```python
# User asks a follow-up question to the ReportAgent
context = client.search(
    "How did public opinion shift between round 1 and round 3?",
    filters={"user_id": project_id},
    top_k=20
)

# Feed context into your LLM prompt for the ReportAgent
report_context = "\n".join([f"- {r['memory']}" for r in context["results"]])
```

## ID Mapping Strategy

Map MiroFish concepts to Mem0 identifiers for clean memory isolation:

| MiroFish Concept | Mem0 Identifier | Example | Purpose |
|---|---|---|---|
| Project / Simulation | `user_id` | `"mirofish_housing_sim"` | Isolates all memories for one simulation |
| Individual Agent | `agent_id` | `"mayor_chen"` | Per-agent memory isolation |
| Simulation Run | `run_id` | `"run_2026_03_17"` | Separates repeated runs of same simulation |
| Action Type | `metadata["action"]` | `"CREATE_POST"` | Filter by action type |
| Platform | `metadata["platform"]` | `"twitter"` | Filter by simulated platform |
| Round Number | `metadata["round"]` | `5` | Filter by simulation round |

<Tip>
  Use `user_id` for project-level isolation and `agent_id` for agent-level isolation. This lets you search across all agents (`filters={"user_id": ...}`) or within a single agent (`filters={"AND": [{"user_id": ...}, {"agent_id": ...}]}`).
</Tip>

## Advanced: Graph Memory with Neo4j

For simulations that need **automatic entity extraction and relationship mining** (closest to Zep's knowledge graph), use Mem0 OSS with Graph Memory:

```python
from mem0 import Memory

config = {
    "llm": {
        "provider": "openai",
        "config": {"model": "gpt-4o-mini", "temperature": 0.1}
    },
    "embedder": {
        "provider": "openai",
        "config": {"model": "text-embedding-3-small"}
    },
    "graph_store": {
        "provider": "neo4j",
        "config": {
            "url": "bolt://localhost:7687",
            "username": "neo4j",
            "password": "password"
        }
    }
}

memory = Memory.from_config(config)

# Ingest seed document — entities and relationships are extracted automatically
memory.add(
    "Mayor Chen supports the zoning reform. Wang opposes it and filed a petition with 3,000 signatures.",
    user_id="mirofish_housing_sim"
)

# Search returns both memories AND extracted relations
results = memory.search("Mayor Chen", user_id="mirofish_housing_sim")

print("Memories:", results.get("results", []))
print("Relations:", results.get("relations", []))
# Relations: [
#   {"source": "Mayor Chen", "relationship": "SUPPORTS", "target": "zoning reform"},
#   {"source": "Wang", "relationship": "OPPOSES", "target": "zoning reform"},
#   {"source": "Wang", "relationship": "FILED", "target": "petition"}
# ]
```

This gives you:
- **Automatic entity extraction** from raw text (names, organizations, policies)
- **Relationship mining** (who supports/opposes/filed what)
- **Graph traversal** for connected entity discovery
- **Combined search** returning both semantic memories and structured relations

<Info>
  Graph Memory requires a running Neo4j instance. For quick local setup: `docker run -p 7687:7687 -e NEO4J_AUTH=neo4j/password neo4j:5`
</Info>

## Verify the Integration

Run these checks to confirm everything works:

```python
from mem0 import MemoryClient
import time

client = MemoryClient()
test_id = "mirofish_verify_test"

# 1. Store a memory
result = client.add(
    [{"role": "user", "content": "Agent_Alice posted about housing costs"}],
    user_id=test_id,
    agent_id="alice"
)
print(f"add() status: {result['results'][0]['status']}")

# 2. Wait for async processing
time.sleep(30)

# 3. Get all memories
all_mems = client.get_all(filters={"user_id": test_id})
print(f"get_all() returned {len(all_mems['results'])} memories")
for m in all_mems["results"]:
    print(f"  - {m['memory']}")

# 4. Search for it
results = client.search(
    "housing costs",
    filters={"AND": [{"user_id": test_id}, {"agent_id": "alice"}]}
)
print(f"search() returned {len(results.get('results', []))} results")

# 5. Cleanup
client.delete_all(user_id=test_id)
print("Verification complete!")
```

<Note>
Memory processing on Mem0 Platform is asynchronous. `add()` returns immediately with a `PENDING` status. Memories become available via `get_all()` within seconds, but `search()` may take up to 30-60 seconds as embeddings are generated and indexed.
</Note>

## Troubleshooting

| Problem | Solution |
|---|---|
| `AuthenticationError` | Set `export MEM0_API_KEY="m0-..."` or pass `api_key` to `MemoryClient()` |
| `ValidationError: Filters are required` | `search()` and `get_all()` require `filters={"user_id": "..."}`. Use `filters` dict, not bare `user_id` kwarg. |
| `search()` returns empty but `get_all()` works | Memory processing is async on Platform. Wait 30-60 seconds after `add()` for search indexing to complete. Use `get_all()` as a fallback for immediate retrieval. |
| Network errors | If self-hosting, set `MEM0_BASE_URL` to your deployment URL |
| OSS mode errors | Confirm `OPENAI_API_KEY` is set and the configured model is accessible |
| Want knowledge graph features | Use Mem0 OSS with `graph_store` configured for Neo4j (see [Advanced section](#advanced-graph-memory-with-neo4j)) |
