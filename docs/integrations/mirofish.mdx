# MiroFish integration

Connect [MiroFish](https://github.com/666ghj/MiroFish) multi-agent swarm intelligence simulations to Mem0 so every agent can persist and recall memory across simulation stages with per-agent isolation.

<Info>
  **Prerequisites**
  - Mem0: `pip install mem0ai`, `MEM0_API_KEY` (or self-hosted endpoint)
  - MiroFish: cloned and set up per [MiroFish README](https://github.com/666ghj/MiroFish)
  - Optional: Neo4j for graph memory (entity/relationship extraction)
</Info>

<Note>MiroFish is a Python-based framework. A TypeScript path is not available yet.</Note>

## Why Mem0 for MiroFish?

MiroFish currently uses Zep Cloud as its sole memory layer. Mem0 offers significant advantages for multi-agent simulations:

| Zep Cloud Limitation | Mem0 Solution |
|----------------------|--------------|
| Cloud-only, no self-hosting | Mem0 OSS runs fully local, no API keys needed |
| 1,000 free credits/month | Mem0 OSS is unlimited; Platform free tier offers 10K memories |
| 2,000 node hard cap | No artificial limits on memory count |
| No per-agent memory isolation | Native `user_id` / `agent_id` / `run_id` partitioning |
| Write-only during simulation | Agents can read their own memories mid-simulation |
| 600s episode processing lag | `add()` is synchronous, memories immediately available |
| Complex pagination boilerplate | Simple `get_all()` and `search()` APIs |

## Store and retrieve agent memories

<Tabs>
  <Tab title="Platform">
```python
from mem0 import MemoryClient

client = MemoryClient()

# Store an agent's action during simulation
client.add(
    "Agent_Alice posted: 'The new policy will increase housing costs significantly'",
    user_id="simulation_001",
    agent_id="agent_alice",
    metadata={"platform": "twitter", "action": "CREATE_POST", "round": 5}
)

# Retrieve agent's memories for report generation
results = client.search(
    "What did Alice say about housing policy?",
    user_id="simulation_001",
    agent_id="agent_alice"
)
for r in results["results"]:
    print(f"- {r['memory']}")
```
  </Tab>

```python
from mem0 import Memory

config = {
```

## How Mem0 maps to MiroFish's workflow

MiroFish runs a five-stage pipeline. Here is how Mem0 plugs into each stage:

| MiroFish Stage | What Happens | Mem0 Feature Used |
|---------------|-------------|-------------------|
| **Graph Building** | Seed documents to entity/relationship extraction | `memory.add()` with graph store extracts entities automatically |
| **Entity Extraction** | Filter entities by type for agent creation | `memory.get_all()` + metadata filtering |
| **Simulation Memory** | Real-time agent actions to memory updates | `memory.add()` per agent with `agent_id` isolation |
| **Report Retrieval** | Semantic search over simulation history | `memory.search()` with filters |
| **Agent Interviews** | Context retrieval for agent responses | `memory.search(agent_id=...)` for agent-specific recall |

## ID mapping strategy

Map MiroFish concepts to Mem0 identifiers for clean memory isolation:

| MiroFish Concept | Mem0 Identifier | Example |
|-----------------|----------------|---------|
| Project / Graph | `user_id` | `"mirofish_a1b2c3d4"` |
| Simulation Run | `run_id` | `"sim_run_2026_03_17"` |
| Individual Agent | `agent_id` | `"agent_alice"` |
| Action Type | `metadata.action_type` | `"CREATE_POST"` |
| Platform | `metadata.platform` | `"twitter"` |
| Round Number | `metadata.round` | `5` |

## Verify the integration

- `memory.add()` returns extracted memories without errors.
- `memory.search()` retrieves relevant agent-specific memories.
- `memory.get_all(agent_id=...)` returns only that agent's memories.
- Mem0 dashboard shows memories scoped to the correct `user_id` and `agent_id`.

## Troubleshooting

- **Missing MEM0_API_KEY** - set `export MEM0_API_KEY="sk-..."` or pass `api_key` into `MemoryClient`.
- **No memories returned** - ensure `agent_id`/`user_id` in your query match what you used when writing.
- **Network errors to Mem0** - if self-hosting, set `MEM0_BASE_URL` to your deployment URL.
