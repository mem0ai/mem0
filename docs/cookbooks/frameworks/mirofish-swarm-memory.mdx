Build a complete multi-agent swarm simulation with persistent memory using Mem0 and [MiroFish](https://github.com/666ghj/MiroFish) patterns. This cookbook walks through all five stages of MiroFish's pipeline — from document ingestion through report generation — with full runnable code.

## Overview

This example implements a **Housing Policy Prediction Simulation** following MiroFish's five-stage workflow:

1. **Graph Building** — Ingest seed documents as background knowledge
2. **Environment Setup** — Generate agent profiles enriched with memory context
3. **Simulation** — Track agent interactions across rounds with per-agent isolation
4. **Report Generation** — Semantic retrieval for analysis and reporting
5. **Deep Interaction** — Query the post-simulation memory for follow-up analysis

Three agents debate a housing policy reform:
- **Mayor Chen** — Policy advocate pushing for zoning reform
- **Wang (Homeowner)** — Opposition leader organizing resistance
- **Professor Li** — Academic providing data-driven analysis

## Complete Implementation

```python
"""
MiroFish Swarm Prediction Simulation with Mem0 Memory

Follows MiroFish's 5-stage pipeline:
1. Graph Building    - Ingest seed documents
2. Environment Setup - Enrich agent profiles from memory
3. Simulation        - Track agent actions per round with agent_id isolation
4. Report Generation - Semantic search for analysis
5. Deep Interaction  - Query post-simulation memory

Run:
    export MEM0_API_KEY="m0-..."
    python mirofish_swarm_memory.py
"""

import os
import time
from mem0 import MemoryClient


# ======================================================================
# MiroFish Agent Action Types (matches OASIS simulation output)
# ======================================================================

MIROFISH_ACTIONS = [
    "CREATE_POST", "LIKE_POST", "DISLIKE_POST", "CREATE_COMMENT",
    "LIKE_COMMENT", "DISLIKE_COMMENT", "REPOST", "QUOTE_POST",
    "FOLLOW", "MUTE", "SEARCH_POSTS", "SEARCH_USER",
]


# ======================================================================
# Simulation Engine
# ======================================================================

class MiroFishSimulation:
    """
    Multi-agent simulation with persistent per-agent memory.

    Each agent gets its own memory space via `agent_id`.
    The simulation project is isolated via `user_id`.
    """

    def __init__(self, project_id: str):
        self.project_id = project_id
        self.client = MemoryClient()
        self.stats = {
            "documents_ingested": 0,
            "activities_recorded": 0,
            "rounds_completed": 0,
        }

    # ------------------------------------------------------------------
    # Stage 1: Graph Building — Seed Document Ingestion
    # ------------------------------------------------------------------

    def ingest_documents(self, documents: list[str]):
        """Ingest seed documents as background knowledge.

        MiroFish equivalent: GraphBuilderService.build_graph()
        Zep equivalent: graph.add_batch() with episode polling

        With Mem0, each document is stored and key facts are
        automatically extracted. No polling needed.
        """
        print("  Ingesting documents...")
        for i, doc in enumerate(documents):
            self.client.add(
                [{"role": "user", "content": doc}],
                user_id=self.project_id,
                metadata={"stage": "graph_building", "source": "seed_document", "chunk_index": i}
            )
        self.stats["documents_ingested"] = len(documents)
        print(f"  Ingested {len(documents)} documents")

    # ------------------------------------------------------------------
    # Stage 2: Environment Setup — Agent Profile Enrichment
    # ------------------------------------------------------------------

    def enrich_agent_profile(self, agent_name: str, persona_query: str) -> list[str]:
        """Search memory for context relevant to an agent's persona.

        MiroFish equivalent: OasisProfileGenerator using graph.search()

        Returns facts that can be injected into the agent's system prompt
        to give it knowledge of the simulation world.
        """
        results = self.client.search(
            persona_query,
            filters={"user_id": self.project_id},
            top_k=10
        )
        facts = [r["memory"] for r in results.get("results", [])]
        print(f"  {agent_name}: found {len(facts)} relevant facts for profile")
        return facts

    # ------------------------------------------------------------------
    # Stage 3: Simulation — Agent Activity Tracking
    # ------------------------------------------------------------------

    def record_action(self, agent_id: str, agent_name: str,
                      action_type: str, content: str,
                      platform: str, round_num: int):
        """Record a single agent action as a memory.

        MiroFish equivalent: ZepGraphMemoryUpdater.add_activity()
        Zep equivalent: graph.add(type="text", data=episode_text)

        Each action is scoped to the agent via `agent_id`,
        preventing memory cross-contamination between agents.
        """
        # Format content the way MiroFish's OASIS engine outputs it
        formatted = f"{agent_name} [{action_type}]: {content}"

        self.client.add(
            [{"role": "user", "content": formatted}],
            user_id=self.project_id,
            agent_id=agent_id,
            metadata={
                "action_type": action_type,
                "platform": platform,
                "round": round_num,
                "agent_name": agent_name,
            }
        )
        self.stats["activities_recorded"] += 1

    def run_round(self, round_num: int, activities: list[tuple]):
        """Execute one simulation round.

        Args:
            round_num: Current round number
            activities: List of (agent_id, agent_name, action_type, content, platform)
        """
        print(f"  Round {round_num}: {len(activities)} actions")
        for agent_id, agent_name, action_type, content, platform in activities:
            self.record_action(agent_id, agent_name, action_type, content, platform, round_num)
        self.stats["rounds_completed"] = max(self.stats["rounds_completed"], round_num)

    def recall_agent_memory(self, agent_id: str, query: str) -> list[str]:
        """Agent recalls its own memories mid-simulation.

        This enables agents to make decisions based on their history.
        Only returns memories for this specific agent (agent_id scoping).
        """
        results = self.client.search(
            query,
            filters={"AND": [{"user_id": self.project_id}, {"agent_id": agent_id}]},
            top_k=5
        )
        return [r["memory"] for r in results.get("results", [])]

    # ------------------------------------------------------------------
    # Stage 4: Report Generation — Semantic Retrieval
    # ------------------------------------------------------------------

    def quick_search(self, query: str, limit: int = 10) -> list[str]:
        """Simple semantic search across all agent memories.

        MiroFish equivalent: ZepToolsService.quick_search()
        """
        results = self.client.search(
            query,
            filters={"user_id": self.project_id},
            top_k=limit
        )
        return [r["memory"] for r in results.get("results", [])]

    def panorama_search(self) -> list[str]:
        """Retrieve all memories for a broad overview.

        MiroFish equivalent: ZepToolsService.panorama_search()
        """
        results = self.client.get_all(filters={"user_id": self.project_id})
        return [r["memory"] for r in results.get("results", [])]

    def agent_search(self, agent_id: str, query: str, limit: int = 10) -> list[str]:
        """Search within a single agent's memory space.

        Returns only memories belonging to the specified agent.
        """
        results = self.client.search(
            query,
            filters={"AND": [{"user_id": self.project_id}, {"agent_id": agent_id}]},
            top_k=limit
        )
        return [r["memory"] for r in results.get("results", [])]

    # ------------------------------------------------------------------
    # Cleanup
    # ------------------------------------------------------------------

    def cleanup(self):
        """Delete all memories for this simulation project."""
        self.client.delete_all(user_id=self.project_id)
        print(f"  Cleaned up all memories for {self.project_id}")


# ======================================================================
# Run the full 5-stage pipeline
# ======================================================================

def main():
    project_id = f"mirofish_housing_{int(time.time())}"
    sim = MiroFishSimulation(project_id=project_id)

    # ==================================================================
    # STAGE 1: Graph Building — Ingest seed documents
    # ==================================================================
    print("=" * 60)
    print("STAGE 1: Graph Building")
    print("=" * 60)

    sim.ingest_documents([
        "The city council proposed a new zoning reform allowing higher "
        "density housing in suburban areas. Mayor Chen expressed strong "
        "support, citing a 40% housing shortage affecting young professionals. "
        "The reform would allow buildings up to 8 stories in previously "
        "restricted 3-story zones.",

        "Local homeowners association president Wang opposes the reform, "
        "arguing it will decrease property values by 15-20%. The association "
        "represents 5,000 homeowners in the affected districts. Wang has "
        "organized three community meetings and collected 2,000 signatures.",

        "Professor Li from Beijing University published research showing "
        "similar reforms in Shenzhen led to 15% price drops in existing "
        "homes but created 30% more affordable housing units within 3 years. "
        "The study covered 12 districts and 50,000 housing units.",
    ])

    # Memory processing is asynchronous on Mem0 Platform.
    # Allow time for memories to be indexed before searching.
    print("  Waiting for memory processing...")
    time.sleep(30)

    # ==================================================================
    # STAGE 2: Environment Setup — Enrich agent profiles
    # ==================================================================
    print("\n" + "=" * 60)
    print("STAGE 2: Environment Setup")
    print("=" * 60)

    mayor_context = sim.enrich_agent_profile(
        "Mayor Chen",
        "housing reform support policy advocacy"
    )
    wang_context = sim.enrich_agent_profile(
        "Wang",
        "homeowner opposition property values community"
    )
    li_context = sim.enrich_agent_profile(
        "Professor Li",
        "academic research housing data Shenzhen"
    )

    # These facts would be injected into each agent's system prompt
    # in MiroFish's OasisProfileGenerator
    print("\n  Example profile context for Mayor Chen:")
    for fact in mayor_context[:3]:
        print(f"    - {fact}")

    # ==================================================================
    # STAGE 3: Simulation — Run agent interactions
    # ==================================================================
    print("\n" + "=" * 60)
    print("STAGE 3: Simulation")
    print("=" * 60)

    # Round 1: Opening statements
    sim.run_round(1, [
        ("mayor_chen", "Mayor Chen", "CREATE_POST",
         "This reform will create 10,000 new housing units by 2028. "
         "Young families deserve affordable homes. #HousingForAll",
         "twitter"),

        ("wang_homeowner", "Wang", "CREATE_POST",
         "Our property values will plummet! The council ignores the "
         "voices of 5,000 homeowners. #StopTheReform",
         "twitter"),

        ("prof_li", "Professor Li", "CREATE_POST",
         "New analysis: Shenzhen zoning data shows net positive outcomes "
         "after 3 years. Short-term pain, long-term gain for housing equity.",
         "twitter"),
    ])

    # Round 2: Debate and interaction
    sim.run_round(2, [
        ("wang_homeowner", "Wang", "CREATE_COMMENT",
         "Replied to Professor Li: 'Shenzhen is a tier-1 city with "
         "completely different dynamics. Your comparison is misleading.'",
         "twitter"),

        ("mayor_chen", "Mayor Chen", "LIKE_POST",
         "Liked Professor Li's post about Shenzhen housing data.",
         "twitter"),

        ("prof_li", "Professor Li", "CREATE_COMMENT",
         "Replied to Wang: 'The methodology controls for city tier "
         "and population density. I invite you to review the full dataset.'",
         "twitter"),

        ("mayor_chen", "Mayor Chen", "CREATE_POST",
         "Data from @ProfLi confirms what we've been saying: zoning "
         "reform works. Let's move forward with evidence, not fear.",
         "twitter"),
    ])

    # Round 3: Escalation and platform expansion
    sim.run_round(3, [
        ("wang_homeowner", "Wang", "CREATE_POST",
         "Filing formal petition with 3,000 signatures against the "
         "zoning reform. Council meeting next Tuesday. All homeowners "
         "must attend!",
         "reddit"),

        ("mayor_chen", "Mayor Chen", "CREATE_POST",
         "Announcing public town hall on zoning reform this Saturday. "
         "All voices welcome. Data-driven decisions benefit everyone.",
         "twitter"),

        ("prof_li", "Professor Li", "CREATE_POST",
         "Published full dataset and methodology on my university page. "
         "Transparency is essential for informed public debate.",
         "twitter"),

        ("wang_homeowner", "Wang", "FOLLOW",
         "Followed @MayorChen to monitor policy updates.",
         "twitter"),
    ])

    # Mid-simulation: agent recalls own memory
    print("\n  Mid-simulation recall for Mayor Chen:")
    mayor_recall = sim.recall_agent_memory(
        "mayor_chen",
        "What positions have I taken on housing reform?"
    )
    for mem in mayor_recall:
        print(f"    - {mem}")

    # Allow Platform indexing time for all activities
    print("\n  Waiting for memory processing...")
    time.sleep(30)

    # ==================================================================
    # STAGE 4: Report Generation — Retrieve memories for analysis
    # ==================================================================
    print("\n" + "=" * 60)
    print("STAGE 4: Report Generation")
    print("=" * 60)

    # Quick search: targeted query
    print("\n  Quick Search: 'opposition to housing reform'")
    opposition = sim.quick_search("opposition to housing reform", limit=5)
    for mem in opposition:
        print(f"    - {mem}")

    # Quick search: another angle
    print("\n  Quick Search: 'data and evidence about housing'")
    evidence = sim.quick_search("data and evidence about housing", limit=5)
    for mem in evidence:
        print(f"    - {mem}")

    # Agent-specific search
    print("\n  Agent Search: Wang's activities")
    wang_activities = sim.agent_search("wang_homeowner", "all actions and statements")
    for mem in wang_activities:
        print(f"    - {mem}")

    # Panorama: full overview
    print("\n  Panorama Search: all memories")
    all_memories = sim.panorama_search()
    print(f"    Total memories: {len(all_memories)}")
    for mem in all_memories[:5]:
        print(f"    - {mem}")
    if len(all_memories) > 5:
        print(f"    ... and {len(all_memories) - 5} more")

    # ==================================================================
    # STAGE 5: Deep Interaction — Post-simulation queries
    # ==================================================================
    print("\n" + "=" * 60)
    print("STAGE 5: Deep Interaction")
    print("=" * 60)

    queries = [
        "How did the debate evolve across the three rounds?",
        "What evidence was cited by each side?",
        "Which agent gained the most engagement?",
    ]

    for query in queries:
        print(f"\n  Query: '{query}'")
        results = sim.quick_search(query, limit=3)
        for mem in results:
            print(f"    - {mem}")

    # ==================================================================
    # Summary
    # ==================================================================
    print("\n" + "=" * 60)
    print("SIMULATION COMPLETE")
    print("=" * 60)
    print(f"  Project ID:        {project_id}")
    print(f"  Documents ingested: {sim.stats['documents_ingested']}")
    print(f"  Activities tracked: {sim.stats['activities_recorded']}")
    print(f"  Rounds completed:  {sim.stats['rounds_completed']}")
    print(f"  Total memories:    {len(all_memories)}")

    # Cleanup (uncomment to delete all memories)
    # sim.cleanup()
```

## How It Works

### Per-Agent Memory Isolation

The key design pattern: `user_id` scopes the simulation project, `agent_id` scopes individual agents.

```python
# Store: agent-scoped memory (add uses top-level kwargs)
client.add(messages, user_id="my_sim", agent_id="mayor_chen", ...)

# Search across ALL agents in a simulation
client.search("housing reform", filters={"user_id": "my_sim"})

# Search within ONE agent's memory only
client.search("housing reform", filters={"AND": [{"user_id": "my_sim"}, {"agent_id": "mayor_chen"}]})

# Get all memories for a simulation
client.get_all(filters={"user_id": "my_sim"})
```

This prevents Wang from accessing Mayor Chen's private reasoning while still allowing cross-agent queries for report generation.

### Stage Mapping

Each MiroFish pipeline stage maps to specific Mem0 operations:

| MiroFish Stage | What Happens | Mem0 Call |
|---|---|---|
| **1. Graph Building** | Ingest seed docs | `client.add(doc, user_id=project)` |
| **2. Environment Setup** | Enrich agent personas | `client.search(query, filters={"user_id": project})` |
| **3. Simulation** | Track per-agent actions | `client.add(action, user_id=project, agent_id=agent)` |
| **3. Simulation** | Mid-round recall | `client.search(query, filters={"AND": [{"user_id": ...}, {"agent_id": ...}]})` |
| **4. Report Generation** | Targeted analysis | `client.search(query, filters={"user_id": project})` |
| **4. Report Generation** | Full overview | `client.get_all(filters={"user_id": project})` |
| **5. Deep Interaction** | Follow-up queries | `client.search(query, filters={"user_id": project})` |

### Zep-to-Mem0 Migration Reference

For developers familiar with MiroFish's Zep integration:

| MiroFish Service | Zep Call | Mem0 Equivalent |
|---|---|---|
| GraphBuilderService | `client.graph.create()` | Implicit on first `add()` |
| GraphBuilderService | `client.graph.set_ontology()` | Not needed (Mem0 extracts entities automatically) |
| GraphBuilderService | `client.graph.add_batch(episodes)` | `client.add()` per chunk |
| GraphBuilderService | `client.graph.episode.get(uuid)` | Not needed (`add()` is synchronous) |
| GraphBuilderService | `client.graph.delete(id)` | `client.delete_all(user_id=...)` |
| ZepEntityReader | `client.graph.node.get_by_graph_id()` | `client.get_all(filters={"user_id": ...})` |
| ZepEntityReader | `client.graph.node.get(uuid)` | `client.search(entity_name, filters={"user_id": ...})` |
| ZepEntityReader | `client.graph.node.get_entity_edges()` | Use Graph Memory with Neo4j for relations |
| ZepGraphMemoryUpdater | `client.graph.add(type="text")` | `client.add(messages, user_id=..., agent_id=...)` |
| ZepToolsService | `client.graph.search(query, scope)` | `client.search(query, filters={"user_id": ...})` |
| ZepToolsService | Panorama (all nodes + edges) | `client.get_all(filters={"user_id": ...})` |
| OasisProfileGenerator | `client.graph.search()` | `client.search(query, filters={"user_id": ...})` |

## Advanced: Graph Memory for Entity Extraction

For simulations that need **knowledge graph capabilities** (entity extraction, relationship mining), use Mem0 OSS with Neo4j:

```python
from mem0 import Memory

config = {
    "llm": {
        "provider": "openai",
        "config": {"model": "gpt-4o-mini", "temperature": 0.1}
    },
    "embedder": {
        "provider": "openai",
        "config": {"model": "text-embedding-3-small"}
    },
    "graph_store": {
        "provider": "neo4j",
        "config": {
            "url": "bolt://localhost:7687",
            "username": "neo4j",
            "password": "password"
        }
    }
}

memory = Memory.from_config(config)

# Ingest — entities and relationships extracted automatically
memory.add(
    "Mayor Chen publicly supported the reform. Wang filed a petition against it "
    "with 3,000 signatures. Professor Li cited Shenzhen data showing 30% more "
    "affordable units after 3 years.",
    user_id="mirofish_housing_sim"
)

# Search returns both memories AND relation triples
results = memory.search("housing reform positions", user_id="mirofish_housing_sim")

for r in results.get("results", []):
    print(f"Memory: {r['memory']}")

for r in results.get("relations", []):
    print(f"Relation: {r['source']} --[{r['relationship']}]--> {r['target']}")
# Relation: Mayor Chen --[SUPPORTS]--> zoning reform
# Relation: Wang --[OPPOSES]--> zoning reform
# Relation: Wang --[FILED]--> petition
# Relation: Professor Li --[CITED]--> Shenzhen data
```

Graph Memory gives you the closest equivalent to Zep's built-in knowledge graph: automatic entity extraction, relationship mining, and graph traversal.

## Best Practices

1. **Unique `user_id` per simulation** — Use timestamps or UUIDs (e.g., `mirofish_housing_1742198400`) to prevent memory collisions between runs
2. **Always set `agent_id`** — Per-agent isolation is the core value prop. Without it, agent memories blend together
3. **Store action type in metadata** — Enables filtering by `CREATE_POST` vs `LIKE_POST` vs `FOLLOW` for structured analysis
4. **Format actions as natural language** — `"Mayor Chen [CREATE_POST]: content"` extracts better than raw JSON
5. **Cleanup old simulations** — Call `delete_all(user_id=...)` when a simulation run is no longer needed
6. **Use Graph Memory for entity work** — If you need to extract entities and relationships (like Zep's knowledge graph), configure Neo4j with Mem0 OSS
