---
title: Tencent Cloud COS Vectors
---

[Tencent Cloud COS Vectors](https://cloud.tencent.com/document/product/436/126985) is a vector storage and retrieval service built on Tencent Cloud COS (Cloud Object Storage). It provides high-performance vector similarity search capabilities for semantic search and AI applications.

### Installation

COS Vectors support requires the Tencent Cloud COS Python SDK. Install it with:

```bash
pip install cos-python-sdk-v5>=1.9.41
```

### Usage

To use Tencent Cloud COS Vectors with Mem0, you need a Tencent Cloud account with the appropriate permissions. Ensure you have your `SecretId`, `SecretKey`, and optionally a `Token` ready.

```python
from mem0 import Memory

config = {
    "vector_store": {
        "provider": "cos_vectors",
        "config": {
            "bucket_name": "my-mem0-vector-bucket",
            "index_name": "my-memories-index",
            "region": "ap-guangzhou",
            "embedding_model_dims": 1536,
            "distance_metric": "cosine",
            "secret_id": "your-secret-id",
            "secret_key": "your-secret-key",
            "token": "your-token",
            "internal_access": False
        }
    }
}

m = Memory.from_config(config)
messages = [
    {"role": "user", "content": "I'm planning to watch a movie tonight. Any recommendations?"},
    {"role": "assistant", "content": "How about a thriller movie? They can be quite engaging."},
    {"role": "user", "content": "I'm not a big fan of thriller movies but I love sci-fi movies."},
    {"role": "assistant", "content": "Got it! I'll avoid thriller recommendations and suggest sci-fi movies in the future."}
]
m.add(messages, user_id="alice", metadata={"category": "movies"})
```

### Config

Here are the parameters available for configuring Tencent Cloud COS Vectors:

| Parameter              | Description                                                                                           | Default Value |
| ---------------------- | ----------------------------------------------------------------------------------------------------- | ------------- |
| `bucket_name`          | The name of the COS Vector bucket to use. It will be created if it doesn't exist.                     | Required      |
| `index_name`           | The name of the vector index within the bucket.                                                       | `mem0`        |
| `region`               | The Tencent Cloud region where the bucket and index reside (e.g., `ap-guangzhou`, `ap-beijing`).      | Required      |
| `embedding_model_dims` | Dimensions of the embedding model. Must match your embedder.                                          | `1536`        |
| `distance_metric`      | Distance metric for similarity search. Options: `cosine`, `euclidean`.                                | `cosine`      |
| `secret_id`            | Tencent Cloud API SecretId for authentication.                                                        | Required      |
| `secret_key`           | Tencent Cloud API SecretKey for authentication.                                                       | Required      |
| `token`                | Tencent Cloud temporary token (optional, used with temporary credentials).                            | `None`        |
| `internal_access`      | Whether to use internal network access. Set to `True` when running within Tencent Cloud VPC.          | `False`       |

### Network Access

COS Vectors supports two access modes:

- **External access** (default): Uses the public endpoint `vectors.<region>.coslake.com`. Suitable for access from outside Tencent Cloud.
- **Internal access**: Uses the internal endpoint `vectors.<region>.internal.tencentcos.com`. Set `internal_access` to `True` when your application runs within the same Tencent Cloud VPC for lower latency and no public network fees.
