[ScyllaDB](https://www.scylladb.com/) is a real-time distributed NoSQL database designed for AI workloads requiring sub-millisecond latency, extreme throughput and high-availability.

### Prerequisites

- A ScyllaDB Cloud cluster
- Python package: `scylla-driver` (install instructions below)

### Installation

pip:
```bash
pip install mem0ai scylla-driver
```

uv:
```bash
uv add mem0ai scylla-driver
```

### Usage

#### ScyllaDB Cloud

```python
import os
from mem0 import Memory

os.environ["OPENAI_API_KEY"] = "sk-xx"

config = {
    "vector_store": {
        "provider": "scylladb",
        "config": {
            "contact_points": ["node-0.aws-us-east-1.xxxxxxxx.clusters.scylla.cloud"],
            "port": 9042,
            "username": "scylla",
            "password": "<your-password>",
            "keyspace": "mem0",
            "collection_name": "memories",
            "datacenter": "AWS_US_EAST_1",   # your ScyllaDB Cloud datacenter
        }
    }
}

m = Memory.from_config(config)
messages = [
    {"role": "user", "content": "I'm looking for a good sci-fi book."},
    {"role": "assistant", "content": "Have you tried Dune? It's a classic."},
    {"role": "user", "content": "I've read Dune. I prefer hard sci-fi with realistic physics."},
    {"role": "assistant", "content": "Then you'll love The Martian or Project Hail Mary by Andy Weir."},
]
m.add(messages, user_id="alice", metadata={"category": "books"})
```


### Config

| Parameter | Description | Default |
| --- | --- | --- |
| `contact_points` | List of ScyllaDB node addresses | Required |
| `port` | CQL native-transport port | `9042` |
| `username` | Database username | `None` |
| `password` | Database password | `None` |
| `keyspace` | Keyspace name | `"mem0"` |
| `collection_name` | Table name for storing vectors | `"memories"` |
| `embedding_model_dims` | Dimensions of the embedding model | `1536` |
| `datacenter` | Local datacenter for DC-aware routing (required for ScyllaDB Cloud) | `None` |
| `use_ssl` | Enable TLS/SSL | `False` |
| `ssl_cert_path` | Path to CA certificate file. Leave `None` to use system CAs. | `None` |


### Setup
1. Sign up at [cloud.scylladb.com](https://cloud.scylladb.com)
2. Create a new cluster
3. In the cluster dashboard, open the **Connect** tab and note your **connection details**:
   - Contact point hostname (e.g., `node-0.aws-us-east-1.xxxxxxxx.clusters.scylla.cloud`)
   - Port: `9042`
   - Datacenter name (e.g., `AWS_US_EAST_1`)
   - Username and password

### How Vector Search Works

Mem0 uses ScyllaDB Vector Search. At table creation time, a vector index is automatically created on the `vector` column:

```sql
CREATE CUSTOM INDEX ON mem0.memories (vector)
    USING 'vector_index'
    WITH OPTIONS = {'similarity_function': 'COSINE'};
```

Searches use the native `ANN OF` syntax for server-side nearest-neighbor retrieval:

```sql
SELECT id, similarity_cosine(vector, ?) AS score, payload
FROM mem0.memories
ORDER BY vector ANN OF ?
LIMIT 5;
```

ScyllaDB returns only the top-K most similar records.

### Performance Considerations

- **DC-aware routing**: Always set `datacenter` to your local datacenter. This ensures the driver routes requests to the nearest shard-owning replica.
- **Replication**: For production, use `NetworkTopologyStrategy`. This is the default.
- **Shard-awareness**: `scylla-driver` automatically routes each CQL request to the shard that owns the partition, eliminating cross-shard hops and reducing latency.
- **Embedding dimensions**: Set `embedding_model_dims` to match your embedding model. This value is baked into the `VECTOR<FLOAT, N>` column definition and cannot be changed without recreating the table.
