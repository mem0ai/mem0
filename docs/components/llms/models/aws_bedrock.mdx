### Setup
- Before using the AWS Bedrock LLM, make sure you have the appropriate model access from [Bedrock Console](https://us-east-1.console.aws.amazon.com/bedrock/home?region=us-east-1#/modelaccess).
- You will also need to authenticate the `boto3` client by using a method in the [AWS documentation](https://boto3.amazonaws.com/v1/documentation/api/latest/guide/credentials.html#configuring-credentials)
- You will have to export `AWS_REGION`, `AWS_ACCESS_KEY`, and `AWS_SECRET_ACCESS_KEY` to set environment variables.

### Usage

```python
import os
from mem0 import Memory

os.environ['AWS_REGION'] = 'us-west-2'
os.environ["AWS_ACCESS_KEY_ID"] = "xx"
os.environ["AWS_SECRET_ACCESS_KEY"] = "xx"

config = {
    "llm": {
        "provider": "aws_bedrock",
        "config": {
            "model": "anthropic.claude-3-5-haiku-20241022-v1:0",
            "temperature": 0.2,
            "max_tokens": 2000,
        }
    }
}

m = Memory.from_config(config)
messages = [
    {"role": "user", "content": "I'm planning to watch a movie tonight. Any recommendations?"},
    {"role": "assistant", "content": "How about thriller movies? They can be quite engaging."},
    {"role": "user", "content": "I’m not a big fan of thriller movies but I love sci-fi movies."},
    {"role": "assistant", "content": "Got it! I'll avoid thriller recommendations and suggest sci-fi movies in the future."}
]
m.add(messages, user_id="alice", metadata={"category": "movies"})
```

### Client timeouts and retries (optional)

By default Mem0 uses botocore's 60-second read timeout, which can be too short for
large memory-extraction calls and cause `Read timeout on endpoint URL` errors. Configure
the boto3 client via three optional keys:

```python
config = {
    "llm": {
        "provider": "aws_bedrock",
        "config": {
            "model": "us.anthropic.claude-sonnet-4-20250514-v1:0",
            # Convenience scalars (seconds):
            "read_timeout": 1000,
            "connect_timeout": 60,
            # Escape hatch for retries / connection pooling / any botocore Config kwarg:
            "boto_client_config": {
                "retries": {"mode": "adaptive", "max_attempts": 5},
                "max_pool_connections": 20,
            },
        }
    }
}
```

- All three keys are **opt-in**; when unset, behavior is unchanged (botocore defaults).
- `boto_client_config` accepts a `botocore.config.Config` object or a dict of its kwargs.
- When both a scalar and `boto_client_config` set the same key, the **scalar wins**.
- **Out of scope:** adaptive throttling/`ThrottlingException` retries are available only through `boto_client_config["retries"]`; an expired STS token mid-call is not handled by these settings.
