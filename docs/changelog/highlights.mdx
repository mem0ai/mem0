<Update label="2026-04-16" description="V3 Pipeline — Post-launch Hardening">

**Hardening pass on the V3 memory pipeline across OSS and Platform.**

- **Platform client endpoints updated** — Python `MemoryClient.add()` now posts to `/v3/memories/add/`; `get_all()` moves to `/v3/memories/` with a paginated envelope (`count` / `next` / `previous` / `results`). Same for the TypeScript client.
- **Default OpenAI model bumped to `gpt-5-mini`** across OSS Python + TypeScript (OpenAI, Azure, OpenAI Structured, LiteLLM fallback).
- **Input validation tightened** — Entity IDs are trimmed and reject empty / whitespace-only strings; `threshold` must be in `[0, 1]`; `top_k` / `topK` default is now `20` (was `100`).
- **`score_breakdown` removed from search responses** in both OSS Python and TypeScript. The top-level `score` remains a `[0, 1]` combined value; per-signal scores are not exposed on the response so the scoring stack is free to evolve.
- **OSS fixes** — FAISS pickle RCE hardened, V3 migration path fixed (entity linking works on Qdrant + Milvus), Qdrant embedded-mode client shared with the entity store, PGVector async init is idempotent, Redis module detection handles both node-redis v3 and v4 response shapes, in-memory `MemoryVectorStore` entity store is fully isolated with backward-compat payload normalization for legacy snake/camel keys.

See the [SDK changelog](/changelog/sdk) for the full patch list.

</Update>

<Update label="2026-04-14" description="Mem0 SDK v2.0.0 / v3.0.0">

**V3 Memory Pipeline — 20+ Point Benchmark Improvements**

Major release of the Mem0 SDKs with a completely rewritten memory pipeline:

- **Python SDK v2.0.0** — Single-pass additive extraction, hybrid search (semantic + BM25 + entity boost), spaCy-based entity extraction and linking, batch operations, message persistence, Valkey cluster mode, graph memory removed from OSS (~4,000 lines across `graph_memory.py`, `memgraph_memory.py`, `kuzu_memory.py`, `apache_age_memory.py`)
- **TypeScript SDK v3.0.0** — V3 OSS pipeline with entity extraction (720+ lines), message persistence, batch embeddings, camelCase parameters, DeepSeek LLM provider, graph memory removed from OSS (1,088 lines across `graph_memory.ts`, `graphs/tools.ts`, `graphs/utils.ts`, `graphs/configs.ts`)
- **Benchmark Results:** LoCoMo 71.4 → 91.6 (+20), LongMemEval 67.8 → 93.4 (+26), BEAM 1M @ 64.1

**Breaking Changes:** `add()` returns ADD-only events, `search()` defaults changed, deprecated parameters removed (`org_id`, `project_id`, `enable_graph`), `limit` → `top_k`/`topK`, **graph memory removed from both Python and TypeScript OSS** (use the Platform API for graph features). See [migration guides](https://docs.mem0.ai/migration/oss-v2-to-v3).

</Update>

<Update label="2026-04-06" description="Mem0 CLI v0.2.2">

**Official Mem0 CLI — Now on PyPI and npm**

A full-featured command-line interface for Mem0, available in both Python and Node.js:

- **Install:** `pip install mem0-cli` or `npm install -g @mem0/cli`
- **Full command suite** — `add`, `search`, `list`, `get`, `update`, `delete`, `import`, `config`, `init`, `status`, `entity`, `event`
- **Interactive setup** — `mem0 init` with email verification or direct API key entry
- **Works everywhere** — Platform (Mem0 Cloud) and self-hosted OSS modes
- **Scriptable** — `--json` flag for CI/CD pipelines and automation
- **Dual SDK** — Same commands, same experience across Python and Node.js

</Update>

<Update label="2026-04-04" description="OpenClaw v1.0.4">

**OpenClaw Plugin — Production-Ready**

The OpenClaw Mem0 plugin went from initial release to production-ready in one week (v1.0.0 → v1.0.4):

- **Skills-based memory architecture** — New extraction pipeline with skill-loader, batched extraction, and domain-aware memory triage
- **Dream gate** — Automatic memory consolidation during idle periods for higher-quality long-term recall
- **Interactive CLI** — `openclaw mem0 init`, `status`, `config`, `import`, and `event` commands
- **Unified tool naming** — `memory_add` and `memory_delete` replace 4 legacy tools, matching the platform API
- **Security hardened** — Path traversal protection, pinned dependencies, 329 tests across 10 files

</Update>

<Update label="2026-04-02" description="Mem0 Plugin for AI Editors">

</Update>
