- **框架与配置**：采用 Mintlify 静态文档生成器，通过 `docs.json` 集中管理导航结构、主题配色（Aspen）、品牌标识及第三方集成（PostHog, Intercom）。
- **内容分层**：逻辑上划分为平台版（Platform）、开源版（Open Source）、API 参考（API Reference）、集成指南（Integrations）及实战手册（Cookbooks）五大核心板块。
- **标准化模板**：在 `templates/` 目录下定义了 API 参考、快速入门、故障排除等 12 种 MDX 内容模板，确保文档结构与交互组件（如 CodeGroup, Card）的一致性。
- **API 规范驱动**：根目录包含 `openapi.json`，作为 API 参考章节的数据源或校验基准，确保接口文档与后端实现同步。
