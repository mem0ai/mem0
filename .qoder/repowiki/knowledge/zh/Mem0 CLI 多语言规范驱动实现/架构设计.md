- **规范驱动开发**：以 `cli/cli-spec.json` 为单一事实来源（Single Source of Truth），定义了所有命令、参数、选项分组及 API 端点，确保跨语言行为一致性。
- **抽象后端隔离**：Node (`backend/base.ts`) 与 Python (`backend/base.py`) 均实现了统一的 `Backend` 抽象接口，将 CLI 逻辑与底层 HTTP 通信解耦。
- **统一配置管理**：两个子模块遵循相同的配置优先级策略（CLI 标志 > 环境变量 > 本地配置文件），并共享 `~/.mem0/config.json` 存储结构。
