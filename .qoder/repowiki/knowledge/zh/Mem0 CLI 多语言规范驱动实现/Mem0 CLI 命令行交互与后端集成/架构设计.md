- **入口与框架**：以 `mem0_cli.app:main` 为 Typer 应用入口，通过 `__main__.py` 支持模块调用。
- **分层架构**：采用“命令层 (`commands/`) - 业务逻辑层 (`backend/`) - 基础设施层 (`config.py`, `output.py`)”的分层设计。
- **接口抽象**：定义 `Backend` 抽象基类 (`backend/base.py`)，目前由 `PlatformBackend` (`backend/platform.py`) 实现，通过 `httpx` 与 SaaS API 通信。
- **配置管理**：使用 `dataclass` 定义配置结构，持久化至 `~/.mem0/config.json`，并支持环境变量覆盖。
- **输出适配**：`output.py` 封装了文本、JSON、表格及 Agent 模式的结构化信封输出，实现展示逻辑与业务逻辑解耦。
