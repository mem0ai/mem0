- 所有命令处理器均遵循“解析参数 -> 获取后端实例 -> 执行业务逻辑 -> 格式化输出”的统一流水线，并通过 `getBackendAndConfig` 统一处理 API Key 校验。
- 支持 `--json` 和 `--agent` 双模输出，在 JSON 模式下自动包裹包含状态、耗时及作用域信息的标准信封（Envelope），便于 LLM 代理解析。
- 实体 ID（user/agent/app/run）解析遵循“显式参数优先于配置文件默认值”的优先级策略，且在存在显式参数时禁止混合默认值以避免过度过滤。
- 敏感配置（如 API Key）在控制台展示时强制进行脱敏处理（仅显示首尾字符），并在写入配置文件时设置严格的文件系统权限（0o600）。
