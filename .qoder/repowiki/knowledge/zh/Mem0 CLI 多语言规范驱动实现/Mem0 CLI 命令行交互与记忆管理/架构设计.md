- **入口与路由**：以 `src/index.ts` 为唯一入口，基于 `commander` 构建命令树，通过动态 `import()` 懒加载 `src/commands/` 下的业务逻辑，实现模块解耦。
- **后端抽象层**：`src/backend/` 定义了统一的 `Backend` 接口（`base.ts`）并封装了平台 API 调用（`platform.ts`），通过工厂模式 `getBackend` 屏蔽底层通信细节。
- **配置与状态管理**：`src/config.ts` 负责 `~/.mem0/config.json` 的持久化与环境变量覆盖；`src/state.ts` 维护全局运行时状态（如 Agent Mode 标识、当前命令上下文）。
- **输出格式化**：`src/output.ts` 集中处理多模态输出（文本、JSON、表格），并根据 `--json` 或 `--agent` 标志自动切换信封格式，确保机器可读性。
- **构建体系**：采用 `tsup` 进行 ESM 打包，通过 `define` 注入版本信息，产物为单文件可执行脚本。
