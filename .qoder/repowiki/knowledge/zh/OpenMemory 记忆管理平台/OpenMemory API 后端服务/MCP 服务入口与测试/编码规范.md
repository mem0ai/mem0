- 所有 MCP 工具函数在执行前必须通过 `get_memory_client_safe()` 获取客户端，并在客户端不可用时返回友好的错误提示而非抛出异常。
- 在 MCP 工具执行过程中，必须通过 `SessionLocal()` 创建独立的数据库会话，并使用 `try...finally` 块确保会话在操作结束后被正确关闭。
- 所有涉及记忆状态变更的操作（如添加、删除）都必须同步创建 `MemoryStatusHistory` 记录以维护审计日志。
- 在处理用户请求时，必须通过 `check_memory_access_permissions` 验证当前应用对目标记忆的访问权限，确保数据隔离。
