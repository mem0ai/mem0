- **页面入口**：`memories/page.tsx` 作为记忆列表页，`memory/[id]/page.tsx` 作为详情页，均采用 Next.js App Router 的客户端组件模式（"use client"）。
- **列表架构**：`MemoriesSection` 负责数据获取与状态管理，组合 `MemoryFilters`（搜索与过滤）、`MemoryTable`（数据展示与行级操作）及分页组件。
- **详情架构**：`MemoryDetails` 展示核心内容，侧边栏集成 `AccessLog`（访问日志）与 `RelatedMemories`（相关记忆），通过 Redux 同步全局状态。
- **交互逻辑**：依赖 `useMemoriesApi` Hook 封装 API 调用，利用 Redux (`memoriesSlice`) 管理选中状态与数据缓存，通过 URL SearchParams 驱动分页与搜索状态。
