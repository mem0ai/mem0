- 入口与导出：`src/index.ts` 统一导出 `createMem0/mem0`、类型定义及记忆工具函数（`addMemories/retrieveMemories/searchMemories/getMemories`）。
- 核心层：`Mem0GenericLanguageModel` 实现 `LanguageModelV3`，在 `doGenerate/doStream` 中先调用 `mem0-utils` 进行记忆写入/检索，再将系统提示拼接到消息头部后委托下游模型执行；当存在记忆时，通过 `providerMetadata.mem0` 附加记忆来源信息。
- 路由层：`Mem0ClassSelector` 根据 `provider` 选择具体后端，内部由 `Mem0AITextGenerator`（`provider-response-provider.ts`）基于 `@ai-sdk/*` 工厂创建对应模型的 `LanguageModelV3` 实例并转发 `doGenerate/doStream`。
- 配置与类型：`mem0-types.ts` 定义 `Mem0ConfigSettings/Mem0ChatSettings/LLMProviderSettings`；`mem0-provider.ts` 提供 `Mem0Provider` 接口与 `createMem0` 工厂，支持 `chat/completion/languageModel` 三种调用形态。
- 依赖方向：上层（Facade/Provider）→ 通用语言模型 → 选择器 → 具体 Provider 封装；记忆读写独立于模型层，通过工具函数与 Mem0 API 交互。
