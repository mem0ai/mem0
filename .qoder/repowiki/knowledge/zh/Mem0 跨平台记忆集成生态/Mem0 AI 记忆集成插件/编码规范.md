- 所有自动化脚本均遵循容错设计原则，通过在命令末尾附加 `2>/dev/null || true` 确保单个环节失败不会中断整个插件的生命周期流程。
- 采用统一的身份解析策略，优先读取 `MEM0_USER_ID` 和 `MEM0_APP_ID` 环境变量，缺失时则回退至操作系统用户名或 Git 远程仓库名称。
- 实施严格的敏感信息过滤机制，在将用户输入或工具输出存入记忆前，通过正则表达式匹配并替换 API Key、Token 等潜在泄露内容。
- 在存储记忆时强制执行元数据标准化，自动补全 `source`、`confidence`、`session_id` 及 `branch` 等字段，以确保后续检索的准确性与可追溯性。
