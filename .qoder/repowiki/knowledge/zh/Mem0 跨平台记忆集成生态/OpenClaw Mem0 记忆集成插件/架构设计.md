- **入口与生命周期**：`index.ts` 作为插件入口，通过 `definePluginEntry` 注册服务、工具（Tools）和 CLI 命令。根据配置动态选择 `PlatformBackend`（云端 API）或 `OSSProvider`（本地 SDK）。
- **后端抽象层**：`backend/` 目录定义了统一的 `Backend` 接口，`providers.ts` 实现了适配器模式，将 Mem0 官方 SDK (`mem0ai`) 的调用标准化，屏蔽了云端与自托管（Qdrant/PGVector/SQLite）的实现差异。
- **技能驱动架构 (Skills Mode)**：模块内置 `skills/` 目录，包含 `memory-triage`（事实提取协议）、`memory-dream`（记忆合并与清理）等 Markdown 协议文件。在激活模式下，插件通过 `before_prompt_build` 钩子注入这些协议，由大模型自主决定记忆的存储与检索策略。
- **模块化设计**：核心逻辑按职责拆分，`tools/` 存放具体的 Agent 工具实现，`cli/` 处理交互式配置向导，`isolation.ts` 负责多智能体环境下的用户 ID 命名空间隔离。
