- 采用 Provider 工厂模式，通过 `createProvider` 根据 `mode` 配置项动态实例化 `PlatformProvider` 或 `OSSProvider`，确保上层业务逻辑与底层存储解耦。
- 实施严格的会话隔离机制，利用 `sessionKey` 和 `agentId` 自动生成唯一的 `userId` 命名空间，防止多智能体场景下的记忆污染。
- 遵循“静默失败”原则，在自动回忆（Auto-recall）和遥测上报等辅助流程中捕获异常并记录日志，确保记忆功能的故障不会阻断主对话流程。
- 统一数据归一化处理，在 `providers.ts` 中通过 `normalizeMemoryItem` 等函数抹平云端 API 与本地 SDK 在字段命名（如 `user_id` vs `userId`）上的差异。
