- **按能力域分层组织**：测试用例按功能边界划分为 `llms/`（大语言模型）、`embeddings/`（嵌入模型）、`vector_stores/`（向量数据库）、`memory/`（核心内存逻辑）及 `rerankers/`（重排序器），每个子目录对应一个特定的技术抽象层。
- **基于 pytest 的隔离架构**：广泛使用 `unittest.mock.patch` 和 `pytest.fixture` 对底层依赖（如 OpenAI 客户端、ChromaDB 实例、SQLite 管理器）进行 Mock，确保单元测试不依赖外部网络或持久化存储。
- **同步/异步双轨验证**：针对 `Memory` 和 `AsyncMemory` 核心类提供对称的测试用例，确保异步环境下的状态管理、错误处理及资源释放逻辑与同步实现保持一致。
- **契约驱动的参数校验**：在 `test_client.py` 和 `test_main.py` 中集中测试 API 入口的参数过滤规则（如禁止顶层 `user_id`、空查询拦截），作为 SDK 行为一致性的第一道防线。
