- 所有涉及外部 I/O 的测试均通过 `patch` 注入 Mock 对象，严禁在单元测试中发起真实的 HTTP 请求或数据库连接。
- 针对核心业务逻辑（如 Memory 增删改查），必须同时覆盖同步 (`Memory`) 和异步 (`AsyncMemory`) 两种实现路径的测试用例。
- 在验证复杂对象交互时，优先使用 `call_args.kwargs` 或 `call_args_list` 断言方法调用的具体参数结构，而非仅检查返回值。
- 对于涉及时间戳、ID 生成或随机数的逻辑，测试中需通过 Mock 固定输出或使用确定性断言（如 `pytest.approx`）以确保结果可复现。
